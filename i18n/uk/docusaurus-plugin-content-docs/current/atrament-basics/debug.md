# Налагодження

Щоб увімкнути налагоджувач, додайте глобальний тег `#debug` в головний Ink файл:

```c title="story.ink"
# title: Ваша історія
# debug
```

Щоб відкрити вікно налагоджувача, двічі натисніть клавішу `~` або клікніть на екранній кнопці налагоджувача під кнопкою меню застосунка.

![Кнопка налагоджувача](/img/doc/debugger-button.png)

## Розділи

### Інформація

Відображає назву файла сценарію, сід генератора випадкових чисел, індекс поточного ходу та поточний шлях в сценарії.

### Глобальні теги

Відображає перелік глобальних тегів сценарію. Невідомі теги будуть виділені.

### Змінні

Відображає поточні значення змінних Ink сценарію. Ви можете відфільтрувати їх за ім'ям.

Щоб відредагувати змінну, клікніть на її значення, відредагуйте його, потім клікніть `>` для збереження, чи `X` для скасування змін.

### Візити

Показує перелік відвіданих шляхів сценарію і кількість відвідувань.

### Функції

Щоб виконати функцію Ink, введіть її назву в поле 'Виконати функцію', потім введіть її аргументи в поле 'Аргументи' і натисніть кнопку `>>>`, щоб побачити результат виконання.

Аргументи мають бути вказані в форматі JSON. Наприклад, щоб виконати функцію:
```c
=== function test_function(name, age)
   Мене звуть {name}, і мені {age} років.
```

введіть ці значення в форму:

| Поле | Значення |
| :--- | :--- |
| Виконати функцію | `test_function` |
| Аргументи | `[ "Пітер Паркер", 18 ]` |

### Навігація

Введіть шлях в поле "Перейти до:" і натисніть кнопку `>>>`. Гра перейде за вказаною адресою і продовжить відтворення звідти.

Приклади формату шляхів можна подивитися в розділі "**Візити**". Шлях до вузлу є його ім'ям: `knot_name`, шлях до стібка включає назву вузлу та стібка, розділені крапкою: `knot_name.stitch_name`.
