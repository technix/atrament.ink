# Публікація

Проєкт на Атрамент може бути опублікований у вигляді одиночного HTML файлу, прогресивного веб-застосунку або застосунку для ПК.

## HTML файл

Щоб опублікувати гру у вигляди одиночного HTML файлу (подібно до веб-експорту Inky або експорту Twine), виконайте цю команду в терміналі:

```bash
npm run build-singlefile
```

Опубліковані файли будуть знаходитися в каталозі `build/singlefile`.

:::tip
Збірка у вигляди одиночного файлу використовує лише системні шрифти для зменшення розміру файла. Якщо ви хочете додати в неї всі шрифти з каталогу `resources/fonts`, скористайтеся такою командою: `npm run build-singlefile -- -- --embed-fonts`.
:::


## Веб-застосунок

Щоб опублікувати гру як [прогресивний веб-застосунок](https://developer.mozilla.org/en-US/docs/Web/Progressive_web_apps), запустіть цю команду в терміналі:

```bash
npm run build-web
```

Файли опублікованого веб-застосунку будуть знаходитися в каталозі `build/web`.

Щоб перевірити, як виглядатиме застосунок, запустіть цю команду:
```bash
npm run preview
``` 
Застосунок відкриється за адресою http://localhost:4173/.

## Застосунок для ПК

:::info
Атрамент використовує [NeutralinoJS](https://neutralino.js.org/) для створення застосунків для ПК.
:::

Щоб створити застосунки для Windows, Linux та macOS, запустіть цю команду в терміналі:

```
npm run build-standalone
```

:::warning
Під час першого запуску ця команда завантажує зборочні файлі NeutralinoJS, тож потрібне з'єднання з інтернетом.
:::

Застосунки для всіх платформ будуть знаходитися в каталозі `build/standalone`. Під час публікації будут також створені ZIP-архіви з застосунками для кожної платформи.
